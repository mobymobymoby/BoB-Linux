[1차 발표 피드백]

- 박수현 멘토님 (질문이라기 보다는 제안임)
	1. 방대한 분야를 담고 있는데 프로젝트가 끝나기 전에 의미있는 산출물이 나올까?
	2. 매뉴얼 같은 경우, 이미 있는 것들인데 어떤 특이점이 있는지? (포괄적 질문)
		→ 초보자들이 매뉴얼을 봤을 때, 이해되지 않는 부분이 있음.
 		한글화를 거침과 동시에 이해가 쉬운 방향으로 제작하려함.
	2-1. 이게 과연 의미있는 작업일까? 얼마나 쉽게, 얼마나 친절하게 할 수 있을까?
		어떻게 할 것인지 예시를 제시했으면 좋았을 것 같다.
	2-2. 다 뻔한 툴이라 이미 매뉴얼이 널렸는데, 그걸 얼마나 잘 해줄 수 있을까?
		→ 트레이닝과 이론과 실습에서 익힐 수 있는 매뉴얼을 동시에 제공하여
		특이점을 가지겠다.

- 이경문 멘토님
	1. OVA 파일은 언제 공개할 예정인가?
		→ 현재까지는 CUI, GUI 환경에서 OVA 파일을 만들어두었음.
		배포를 어떻게 할지는 한번 더 생각해봐야 할 것 같다.
	1-1. SDLC(Software Development Life Cycle) 라이프 사이클을 돌아보고,
		2차에서 보완하고, 3차에서 보완하는 방식으로 하면 좋을 것.
	2. 1차 발표에서 OVA 파일을 줬으면 더 전달될 것 같음.
		2차 때는 미완성이라도 해당 OVA 파일을 공개를 해주었으면 좋겠다.
		이 리눅스가 나오면 BOB 멘토님이 가장 먼저 써볼 것이기 때문에 어필을
		 잘하자.  기능이 어떻게 구성되어 있는지는 설치를 해봐야 알 것 같다.
		(칼리 리눅스에서 유튜브 띄워놓고 공부하는 것 보다 더 잘 만들어야 할 듯)

- 전상현 멘토님
	1. 유튜브 보다 어떻게 더 편한지 알고 싶다. 무엇을? 어떻게?
		→ 기본 명령어에 대한 트레이닝, ls 명령어를 예로 해당 명령어의 단서를 
			주고 C언어의 system 함수를 사용해 실제 디렉터리를 보여주는
			그런 트레이닝을 생각 중이다.

=============================================================================
결론 : 칼리 리눅스에서 유튜브 띄워놓고 공부하는 것 보다 더 학습효과가 높아야 함.
(유튜브는 영상 매체고, 저희는 문서 매체라서 학습 시간을 줄일 수 있을 것 같다는 개인적인 생각)
